This epic focuses on long-term infrastructure and program planning, which is essential as modern mosques increasingly transition into multi-functional community centers.
Here are the detailed requirements for Epic 6 to hand over to your development team:
1. Public-Facing "Mosque Roadmap" Dashboard

- The system must include a public visual dashboard that illustrates the mosque's development journey to foster a shared vision and encourage fundraising within the congregation.
- The Past-Present-Future Paradigm: The roadmap should be structured chronologically to demonstrate institutional progress alongside future aspirations.
- Visual Elements: The interface should use goal thermometers, progress bars, and budget usage indicators. For example, the dashboard can display Past successes (e.g., a completed roof repair), Present active needs (e.g., a Ramadan tent), and Future aspirations (e.g., a parking lot expansion or a clinic).

2. Admin Project Setup & Milestone Tracking

- Behind the scenes, your administrators need robust tools to plan and execute these initiatives.
- Project Setup: The system must allow staff to initiate projects by entering essential details, including the project name, designated fund source, estimated cost, start and end dates, and the specific project in-charge.
- Milestone Management: Admins must be able to break large capital campaigns (like building a new prayer hall) into smaller, measurable milestones. The tool should allow them to update completion percentages and track statuses in real-time, which helps identify potential risks or bottlenecks early in the project lifecycle.

3. Project Financial Tracking & Analytics

- To ensure transparency, project management must be tightly integrated with your financial data.
- Transaction Tracking: The module must record and track all receipts and payments tied specifically to each project, providing a clear financial overview.
- Analysis Reports: The system should generate detailed analytical reports that offer insights into a project's progress, cost deviations, and overall fund utilization to help management make informed decisions.

4. Project Completion Workflow

- Formal Closure: The system must facilitate a formal project closure process. Once a project is finished, the module should verify all related receipts and payments, document any delays that occurred, and securely store the comprehensive record for future accountability and transparency.
- By breaking projects down into these measurable steps and publicly celebrating small wins, your mosque leadership can maintain community momentum for long-term capital campaigns.
